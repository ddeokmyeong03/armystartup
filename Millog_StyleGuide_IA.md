# Millog — 디자인 스타일가이드 & 정보구조도 (IA)

> Codespace 개발 핸드오프 문서 · MVP 기준
> 서비스명 `Millog`는 작업용 표기 — 확정 시 일괄 치환
> 최종 갱신 기준: 본 문서 = Single Source of Truth (디자인 결정은 여기서 관리)

---

## 0. 개요

### 0.1 서비스 한 줄 정의
군 장병의 자기계발을 **정형 데이터로 기록**하고, **팀 챌린지**로 지속시키며, 전역 시점에 **성장 보고서**로 증명하는 모바일 서비스.

### 0.2 디자인 원칙
- **신뢰 우선(Credible)** — 핵심 산출물이 '증빙'이다. 화려함보다 *정직하고 읽기 쉬운* 데이터 표현이 먼저다.
- **성장의 가시화(Growth-visible)** — 누적·추세·달성이 한눈에 보여야 한다. 숫자와 진척이 주인공.
- **가볍게 자주(Low-friction)** — 매일 기록하는 앱이다. 입력 마찰 최소화, 탭 수 최소화.
- **절제된 활력(Calm energy)** — 챌린지엔 동기가 필요하나, 사행성·과장 연출은 금지(정직성 규칙 §8).
- **Toss풍 미니멀** — 앱인토스 검증 채널과 톤을 맞춘다. 평면, 여백, 단색.

### 0.3 스타일가이드 선정 사유
- **성장 그린**: '성장'이라는 핵심 가치 + Millog 헤리티지 색감과 정합. 군 친화적이되 소비자 앱답게 산뜻함.
- **라이트 모드 우선**: 매일 쓰는 기록·증빙 앱은 가독성이 1순위. 앱인토스 환경과도 일관. (다크 토큰 동봉)
- **Pretendard**: 국문 가독성 표준, 기존 팀 자산.

### 0.4 톤 & 보이스(마이크로카피)
- 격려하되 단정하지 않음: "성장하고 있어요" (O) / "당신은 완벽합니다" (X)
- 정직 라벨 필수: AI 생성물엔 "AI 초안", 미검증엔 "자기보고", 시안엔 "더미 데이터"
- 문장: 간결·평서. 군대식 명령조 금지.

## 4. 컴포넌트 가이드

모바일 기준 터치 타깃 최소 `44×44`.

### 4.1 버튼
| 종류 | 배경 | 텍스트 | 보더 | 상태 |
|---|---|---|---|---|
| Primary | `brand-500` | `text-on-brand` | none | pressed `brand-600` / disabled `gray-200`+`text-tertiary` |
| Secondary | transparent | `text-primary` | `1px gray-200` | hover bg `gray-50` |
| Ghost/Text | transparent | `brand-600` | none | — |
| Danger | `danger` | `#FFFFFF` | none | 파기 등 파괴적 액션 한정 |
공통: 높이 `48`, 라디우스 `radius-md`, 좌우 패딩 `20`, 폰트 `15/600`.

### 4.2 인풋 / 셀렉트
높이 `48`, 라디우스 `radius-md`, 보더 `1px gray-200`, 배경 `bg-base`.
focus: 보더 `brand-500` + ring `0 0 0 3px rgba(29,158,117,.15)`. placeholder `text-tertiary`.

### 4.3 카드
배경 `bg-elevated`, 보더 `0.5px border`, 라디우스 `radius-lg`, 패딩 `16`. 평면(e0~e1).

### 4.4 메트릭 카드 (성장 요약)
배경 `bg-subtle`, 라디우스 `radius-md`, 패딩 `14~16`.
구조: 라벨 `caption/text-secondary` → 값 `stat/text-primary` → 보조 `micro/text-tertiary`.

### 4.5 배지 / 칩 (정직성 핵심 — §8 참조)
| 배지 | 배경 | 텍스트 | 텍스트내용 |
|---|---|---|---|
| 검증됨 | `brand-50` | `brand-700` | `✓ 검증` |
| 자기보고 | `gray-100` | `gray-600` | `자기보고` |
| AI 초안 | `info` 10% | `info` | `AI 초안` |
| 시안/더미 | `warning` 12% | `warning` | `더미 데이터` |
| 카테고리 칩 | `gray-50` | `gray-600` | 자격증/어학/독서/운동 |

### 4.6 하단 탭 네비게이션
높이 `56` + safe-area. 4탭. 아이콘 `24` + 라벨 `micro`.
active: `brand-primary`, inactive: `gray-400`.
탭: 홈 / 기록 / 챌린지 / 마이.

### 4.7 리스트 로우
최소 높이 `56`, 패딩 `12~16`. (선행 아이콘) + 타이틀 `body` + 후행 메타 `caption/text-tertiary`.

### 4.8 챌린지 카드
헤더(타이틀 + 카테고리 칩) / 진행 표시(랭킹·합불·비경쟁 트랙) / 기간·인원 / CTA.
판정 방식별 색: 합불=`success`/`gray`, 랭킹=`brand`, 비경쟁=`info`.

## 6. 정보구조도 (IA) — MVP

### 6.1 Depth 0 — 온보딩 (탭 외부 · 최초 1회)
```
온보딩
 ├ 목표 설정 (측정가능 목표 1개)
 └ 프로필 입력 (계급·부대·전역일)   # 전역일 = 보고서 트리거(필수)
```

### 6.2 Depth 1 — 하단 탭 (4)
```
홈   ·   기록   ·   챌린지   ·   마이
```
> 진로 탭은 MVP 제외(추천·진로그래프 = Phase 2).

### 6.3 Depth 2~4 — 탭별 화면 (★ = 유료 전환 지점)
```
[홈]
└ 성장 대시보드
    ├ 목표 진척 요약
    ├ 누적 기록 요약 (카테고리별 추세)
    └ 진행 중 챌린지 요약 → [챌린지 상세]

[기록]
└ 기록 메인 (카테고리 탭 + 타임라인)
    ├ 카테고리별 기록 목록 (자격증/어학/독서/운동)
    ├ 기록 입력 폼 (정형 입력)
    │   └ 증빙 첨부 모달 (선택)
    └ 기록 상세 / 수정·삭제

[챌린지]
└ 챌린지 메인 (탐색 + 내 챌린지)
    ├ 챌린지 상세
    │   ├ 참여하기              ★ 유료 챌린지 = 참가비 결제
    │   ├ 참여 현황 (랭킹/합·불/비경쟁 트랙)
    │   ├ 판정 제출 (증빙 업로드)
    │   ├ 응원·댓글
    │   └ 보상 수령 (달성 시)
    └ 챌린지 개설 폼 (카테고리·기간·시험회차·판정방식)
        └ 보상형 설정          ★ 참가비/상금 설정 = 결제 연결

[마이]
└ 마이 메인 (프로필 카드)
    ├ 목표 설정/관리
    ├ 프로필 편집 (계급·부대·전역일)
    ├ 결제 내역 (유료 챌린지 사용 시)
    └ 설정 (알림·보안·로그아웃)
```

## 9. 정직성 / 표기 규칙 (UI에 강제)

증빙 신뢰성이 제품의 생명이므로, 다음 라벨은 **코드 레벨에서 강제**한다.

- **검증 구분**: 모든 기록/성취에 `verified: boolean`. UI는 검증=`success` 체크 아이콘+`검증` 배지, 미검증=`gray` 빈 원+`자기보고` 배지. 둘을 시각적으로 항상 구분.
- **AI 생성물**: LLM이 만든 모든 텍스트(성장 서사 등)에 `AI 초안` 배지 + "본인 편집 가능" 명시.
- **시안/더미**: 비프로덕션 데이터엔 `더미 데이터` 배지(warning).
- **월권 금지 카피**: 보고서는 "성장 기록 증명"이지 자격 "인증/검수 인증서"가 아니다 — 인증/면책 뉘앙스 문구 금지.
- **과장어 금지**: "유일한·완벽한·100%" 등 단정형 회피.

## 10. 핸드오프 노트 (Codespace)

- **스택 가정**: 프론트 React Native(기존 Millog 자산 재활용) 기준. 앱인토스 검증 채널은 WebView/미니앱 병행 가능 — 디자인 토큰은 동일 적용.
- **모드**: 라이트 우선, 다크 토큰 동봉. `prefers-color-scheme` 대응 필수.
- **반응형**: 모바일 우선(폭 360~430). 한 손 조작 고려, 주요 CTA 하단 배치.
- **접근성**: 터치 타깃 ≥44, 본문 대비 ≥4.5:1, 색만으로 정보 전달 금지(검증 표기는 아이콘+텍스트 병행).
- **숫자 포맷**: 화면 노출 수치는 반올림/`toLocaleString`. 통계는 tabular-nums.
- **국문 타이포**: `word-break: keep-all`, 줄바꿈 어색함 방지.
- **토큰 우선**: 하드코딩 색상 금지, 본 문서 CSS 변수만 사용.
