# Millog (밀로그) 사업계획서

> 병사의 군 일과와 자기개발 목표를 기반으로
> 장병e음·국방전직교육원·K-MOOC 강의를 AI가 맞춤 추천하는 서비스

---

## 1. 문제 정의

### 현황과 문제점

| 현황 | 문제 |
|------|------|
| 장병e음·국방전직교육원·K-MOOC 등 국방 자기개발 인프라 존재 | 병사들의 낮은 인지도 및 활용률 |
| 병사는 하루 일과가 정해져 있어 자투리 시간이 불규칙함 | 어느 강의를 언제 들어야 할지 안내 부재 |
| 자기개발 의지는 있으나 목표-강의 연결 방법 모름 | 강의 탐색에 시간 소비, 포기율 높음 |
| 국방부 차원에서 자기개발 사업 지원 확대 중 | 사업 활용률이 낮아 예산 효율 저하 |

---

## 2. 솔루션: Millog

**"목표를 입력하면, AI가 맞는 강의를 찾아드립니다."**

병사가 자신의 목표(토익, 자격증, 코딩 등)와 군 일과를 등록하면,
AI가 장병e음·국방전직교육원·K-MOOC에서 적합한 강의를 골라 추천합니다.

### 핵심 차별점

1. **국방 자기개발 사업 연계 특화** — 민간 앱이 아닌, 장병e음 등 군 전용 플랫폼 중심 추천
2. **군 일과 기반 맞춤 추천** — 기상시간, 취침시간, 가용 학습 시간, 근무 강도 반영
3. **목표 → 강의 자동 매칭** — 목표 등록만 하면 AI가 관련 강의 자동 선별

---

## 3. 대회 어필 포인트

> "병사가 본인의 일과와 자기개발 목표에 한 걸음 더 다가갈 수 있도록
> 국방 자기개발 사업 관련 서비스 및 강의를 추천하여
> 국방 자기개발 사업을 활성화할 수 있는 것을 목표로 삼는다."

- 기존 국방 자기개발 인프라(장병e음, 국방전직교육원)를 **AI로 연결**
- 병사 개개인에게 맞는 강의를 **자동 추천**하여 접근성 향상
- 자기개발 사업 **참여율 증가** → 국방 자기개발 사업 활성화 기여

## 5. 서비스 플로우

```
[회원가입 / 로그인]
        ↓
[목표 등록]  예: 토익 900점 목표, 컴활 자격증 취득
        ↓
[군 일과 등록]  예: 월~금 06:00~22:00 근무, 매일 21:00 점호
        ↓
[프로필 설정]  예: 기상 06:30, 취침 23:00, 가용 학습 시간 60분
        ↓
[AI 강의 추천]  ← 목표 + 가용 시간 + 학습 강도 분석
   ┌─────────────────────────────────────────────────────┐
   │ 장병e음 토익 집중과정 (60분)                        │
   │ "귀하의 토익 목표에 최적화된 RC/LC 전 영역 강의..." │
   ├─────────────────────────────────────────────────────┤
   │ 장병e음 컴퓨터활용능력 2급 (90분)                   │
   │ "자격증 취득 목표와 직접 연결되는 핵심 과정..."     │
   └─────────────────────────────────────────────────────┘
        ↓
[강의 저장 / 바로가기 (장병e음 이동)]
        ↓
[AI 채팅으로 추가 상담]  예: "토익 파트별 공략법 알려줘"
        ↓
[오늘의 계획 확인 / 완료 처리]
```

## 6. 기술 스택

### 백엔드

| 항목 | 기술 | 선정 이유 |
|------|------|-----------|
| Framework | NestJS 10 (Node.js + TypeScript) | ~128MB 경량 실행, Render 무료 플랜 적합 |
| ORM | Prisma 5 | 타입 안전, 마이그레이션 자동화 |
| Database | PostgreSQL 16 | Render 기본 제공 DB |
| Auth | JWT (passport-jwt) | 무상태 인증, 모바일 친화적 |
| AI | OpenAI GPT-4o-mini | 저비용 고품질 추천 이유 생성 |
| API Docs | Swagger | 개발 생산성 향상 |

## 7. AI 추천 알고리즘

```
입력
  ├─ 사용자 활성 목표 (GoalType: STUDY, CERTIFICATE, CODING...)
  └─ 가용 학습 시간 (availableStudyMinutes)

1단계: 규칙 기반 필터링
  ├─ 목표 유형과 일치하는 강의 우선 선별
  ├─ 보편 강의 포함 (targetGoalType = null)
  └─ 강의 시간 ≤ 가용 시간 + 30분 (여유분)

2단계: GPT-4o-mini 추천
  ├─ 후보 강의 목록 + 사용자 컨텍스트 전달
  ├─ 최대 3개 강의 선택 + 추천 이유 생성 (2~3문장)
  └─ JSON 응답 파싱 → DB 저장

3단계: Fallback (GPT 실패 시)
  └─ 규칙 기반 상위 3개 자동 선택 (서비스 연속성 보장)

출력
  └─ CourseRecommendation DB 저장 → 추천 이유 + 우선순위 포함 응답
```
